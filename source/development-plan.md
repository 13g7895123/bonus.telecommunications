## 2. 開發原則與分期策略

### 分期邏輯

本平台依「業務可上線」為切割原則，分為三個可獨立交付的版本：

| 版本 | 代號 | 目標 |
|------|------|------|
| Phase 1 — MVP | v0.1 | 可人工上架商品 + 用戶付款 + 手動開通門號，能開始收第一筆單 |
| Phase 2 — 核心自動化 | v0.5 | 全自動化付款確認 + 開通、代理系統、競拍、機器人通知 |
| Phase 3 — 完整平台 | v1.0 | 增值服務 + 爬蟲監控 + 風控自動化 + 錢包輪替控制台 |

### 技術優先序

1. **先建基礎設施**（資料庫、認證、部署環境），避免後期大規模返工
2. **先通支付鏈路**（TRC20 收款確認 → 訂單開通），此為核心業務價值
3. **UI 後行**：後台先出 API，前端介面在功能可用後再迭代優化

## 6. 關鍵路徑分析

以下路徑上的任意一項延誤，都會直接推遲整體上線日期：

```
1.1（腳手架）
  └─▶ 1.2（開發環境）
        └─▶ 1.4（資料庫 Schema）
              └─▶ 1.5（認證服務）
                    └─▶ 1.9（訂單 + 付款確認）
                          └─▶ 2.1（供應商 API）
                                └─▶ 2.2（自動開通）
                                      └─▶ 3.1（歸集腳本）
                                            └─▶ 3.2（輪替控台）
                                                  └─▶ 3.15（正式上線）
```

**關鍵路徑上的高風險任務：**

| 任務 | 風險原因 | 緩解策略 |
|------|---------|---------|
| 2.1 — eSIM 供應商 API 接通 | 供應商文件可能不完整，沙盒環境申請需時 | Phase 1 期間即開始申請帳號 |
| 1.7 — Tangem xPub 整合 | 官方 SDK 文件較少，需自行研究 | 提前 PoC（概念驗證）|
| 1.8 — TronGrid Webhook | 鏈上事件延遲偶有波動 | 加入重試機制與手動補單工具 |
| 3.1 — TRC20 歸集腳本 | Energy 代付邏輯較複雜 | 預留 2 週緩衝 |

## 7. 風險登記冊

| 風險編號 | 風險描述 | 發生機率 | 影響程度 | 整體等級 | 緩解措施 |
|---------|---------|---------|---------|---------|---------|
| R-01 | eSIM 供應商 API 文件不完整 / 沙盒取得困難 | 高 | 高 | 🔴 嚴重 | 同時洽談 2–3 家，留備案 |
| R-02 | TronGrid Webhook 偶發遺漏事件 | 中 | 高 | 🟠 高 | 加入鏈上輪詢（Fallback）機制 |
| R-03 | Tangem xPub SDK 整合複雜度超預期 | 中 | 中 | 🟠 高 | 提前 PoC，若卡住可改用純軟體 HD Wallet |
| R-04 | 核心成員離職 / 請假 | 低 | 高 | 🟡 中 | 維持文件完整性，避免知識孤島 |
| R-05 | WhatsApp Business API 審核未通過 | 中 | 低 | 🟡 中 | Phase 2 先以 TG Bot 為主，WA 延至 Phase 3 |
| R-06 | 供應商 API 回應速度不穩（>3 秒）| 中 | 中 | 🟡 中 | 加入非同步佇列，開通流程改為後台 Job |
| R-07 | 爬蟲被目標網站封鎖 | 中 | 低 | 🟢 低 | 分散爬蟲 IP、加 User-Agent 輪替、降低頻率 |
| R-08 | TRC20 鏈上 Energy 不足導致歸集失敗 | 低 | 中 | 🟢 低 | 監控 Energy 餘額，低於閾值自動購買 |
| R-09 | 法規環境變化（目標部署地）| 低 | 高 | 🟡 中 | 優先部署 2 個地區節點，保持可快速遷移 |

## 8. MVP 最小可行版本定義

**MVP（Phase 1 結束）上線的最低條件 Checklist：**

### 必須完成（Must Have）

- [ ] 用戶可註冊、登入
- [ ] 管理員可手動上架商品（至少 1 種商品類型）
- [ ] 用戶可下單，系統產生 USDT 收款地址
- [ ] TronGrid Webhook 可偵測入帳，更新訂單狀態
- [ ] 管理員後台可一鍵開通 / 關閉門號
- [ ] 基本訂單列表 + 訂單詳情頁
- [ ] HTTPS 上線，JWT 認證無明顯安全漏洞
- [ ] 幣安 API 匯率換算正常運作

### 暫緩（Phase 2+ 再做）

- 供應商 API 自動開通
- 代理系統
- 競拍
- 機器人通知
- 多語系
